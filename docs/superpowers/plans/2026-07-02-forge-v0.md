# Forge v0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Forge v0 — a local-first Rust engine for justified, perishable decisions (Markdown files as sole truth, derived in-memory graph, premise-staleness propagation, semantic de-dup) exposed through seven MCP tools.

**Architecture:** Cargo workspace with `forge-core` (pure library: record layer → discovery → linker → graph → judge → snapshot, plus embedder seam, recall, guardian), `forge-inspect` (dev probe binary), and `forge-mcp` (thin `rmcp` stdio server). Files are never mutated except by the guardian; the snapshot is an immutable value rebuilt synchronously after every write.

**Tech Stack:** Rust (stable), serde + serde_yaml (frontmatter), toml, walkdir, chrono, sha2, thiserror; Phase 2 adds ort (ONNX Runtime), tokenizers, hf-hub behind an `onnx` feature; Phase 4 adds rmcp + tokio.

**Authoritative documents:** `docs/superpowers/specs/2026-07-02-forge-v0-design.md` (the design; resolves all open decisions) and `forge-v0-build-spec.md` (the parent spec). If this plan and the design disagree, the design wins. **The MCP surface is exactly seven tools** (design §4.I); `supersede` is not a tool — it is `propose_decision` with `supersedes` + `commit`.

**Ground rules for every task:**
- TDD: write the test, watch it fail, implement, watch it pass, commit. See superpowers:test-driven-development.
- Never write derived state into record files. The guardian is the only file writer.
- Corpus problems (bad YAML, collisions, dangling refs, cycles) are **diagnostics data**, never panics/errors.
- Crate versions below are best-known; if a version doesn't resolve, take the latest compatible from crates.io and check docs.rs — do not change the API shape the plan defines.
- Run `cargo fmt` before each commit; keep `cargo clippy --all-targets` clean (warnings allowed only with a stated reason).

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Parsed { Decision(Decision), Force(Force), Error(ParseError) }

#[derive(Debug, Clone, PartialEq)]
pub struct Decision {
    pub id: String, pub title: String,
    pub status: DecisionStatus,           // Proposed|Accepted|Rejected|Deprecated
    pub date: String,                     // ISO-8601 string; not interpreted in v0
    pub cites: Vec<String>, pub supersedes: Vec<String>, pub relates: Vec<String>,
    pub anchors: Vec<serde_yaml::Value>,  // RESERVED: opaque, never resolved
    pub tags: Vec<String>,
    pub body: String, pub path: PathBuf,
}

#[derive(Debug, Clone, PartialEq)]
pub struct Force {
    pub id: String, pub title: String,
    pub depends_on: Vec<String>,          // YAML key: dependsOn
    pub status_log: Vec<StatusEntry>,     // non-empty, validated ordering
    pub superseded_by: Option<String>,    // YAML key: supersededBy
    pub tags: Vec<String>,
    pub body: String, pub path: PathBuf,
}
impl Force { pub fn current_status(&self) -> ForceStatus { self.status_log.last().unwrap().status } }

pub struct StatusEntry { pub status: ForceStatus, pub since: String }
pub enum ForceStatus { Holds, Changed, Retired }   // serde rename lowercase
```

Parsing: split frontmatter (`---\n` … `\n---`), `serde_yaml` into intermediate serde structs (with `#[serde(rename)]` for camelCase keys, `deny_unknown_fields` **off** — unknown keys are tolerated, files are the user's), then validate: id non-empty, status_log non-empty + legal forward-only transitions (`holds→changed`, `holds→retired`, `changed→retired`; equal repeats rejected). Every validation failure → `Parsed::Error { path, message }`. Body = everything after closing delimiter, verbatim.

Shared test helper `fixture(rel: &str) -> PathBuf` goes in `record.rs` tests for now; Task 11 moves it to `tests/fixtures.rs` for integration tests.

```rust
pub struct FallenPremise { pub force_id: String, pub status: ForceStatus, pub distance: u32, pub since: String }
pub enum PremiseVerdict { Fresh, Stale { fallen: Vec<FallenPremise> } }  // fallen sorted by (status desc, distance, id)
pub struct Verdicts {
    pub premise: HashMap<String, PremiseVerdict>,  // decision id -> verdict

    pub frontier: Vec<String>,                     // sorted decision ids
}
pub fn judge(graph: &Graph) -> Verdicts
```
Algorithm: for each force with `current_status() != Holds`, BFS **upward**: level 0 = the fallen force; from any force at level k, forces in `depended_by` are at k+1 and decisions in `cited_by` are at k+1. Superseded = union of all `supersedes` targets + decisions with status Deprecated. Frontier = decisions not superseded.
